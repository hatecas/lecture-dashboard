## 1. 프로젝트 개요

온라인 강의 비즈니스를 위한 **올인원 운영/분석 대시보드**.
강사별 매출, 광고 성과, 고객 응대, 강의 AI 분석까지 하나의 플랫폼에서 처리한다.

| 항목 | 내용 |
|------|------|
| **프론트엔드** | Next.js 16 + React 19 + Tailwind CSS 4 |
| **백엔드** | Python FastAPI + Node.js API Routes |
| **DB** | Supabase (PostgreSQL) + Storage |
| **AI** | Google Gemini 2.0 Flash + Anthropic Claude Sonnet 4 |
| **배포** | Vercel (프론트) + Railway (백엔드) |
| **코드 규모** | 프론트 ~8,000줄 + 백엔드 ~530줄 + API 라우트 20개+ |

## 3. 기능별 작동 원리

### 3-1. 강의 분석 파이프라인 (핵심 기능)

```
사용자가 YouTube URL 입력
         │
         ▼
┌─ Vercel 서버리스 함수 ─────────────────────────┐
│  /api/lecture-analyze-gemini                    │
│  • Bearer 토큰 검증                              │
│  • HuggingFace Space 슬립 모드 감지 & 자동 깨우기    │
│  • 30분 타임아웃 설정 (undici Agent)               │
│  • FormData를 Python 백엔드로 프록시               │
└────────────────┬───────────────────────────────┘
                 │ HTTP POST (FormData)
                 ▼
┌─ Railway Python 백엔드 ────────────────────────┐
│  FastAPI /api/analyze                          │
│                                                │
│  ① 영상 ID 추출 (정규식 — 6가지 URL 패턴 지원)     │
│  ② YouTube 메타데이터 수집 (oEmbed + 페이지 스크래핑) │
│                                                │
│  ┌─ STAGE 1: 자막 추출 (95% 확률) ──────────┐  │
│  │ youtube-transcript-api                    │  │
│  │ 우선순위: 한국어 → 영어 → 기타             │  │
│  │ 50자 이상이면 성공 처리                     │  │
│  │                                           │  │
│  │ 자막 < 10만자: 단일 Gemini 호출            │  │
│  │ 자막 > 10만자: Map-Reduce                 │  │
│  │   ├ 10만자 단위 분할 (문장 경계)            │  │
│  │   ├ Map: 청크별 분석 (3개씩 배치)           │  │
│  │   └ Reduce: 전체 종합 분석                 │  │
│  └───────────────────────────────────────────┘  │
│           │ 실패 시                              │
│           ▼                                     │
│  ┌─ STAGE 2: Gemini 직접 URL (4% 확률) ────┐  │
│  │ YouTube URL을 Gemini에 직접 전달          │  │
│  │ mime_type="video/mp4"                    │  │
│  │ Gemini가 영상 직접 시청 & 분석              │  │
│  └───────────────────────────────────────────┘  │
│           │ 실패 시                              │
│           ▼                                     │
│  ┌─ STAGE 3: yt-dlp 오디오 다운로드 (1%) ───┐  │
│  │ 4가지 우회 전략 순차 시도:                  │  │
│  │  1. Android + skip_webpage               │  │
│  │  2. iOS                                  │  │
│  │  3. Android                              │  │
│  │  4. Web Creator                          │  │
│  │                                           │  │
│  │ 설정: MP3 64K / 타임아웃 300초              │  │
│  │       UA 스푸핑 / IPv4 강제                 │  │
│  │                                           │  │
│  │ → Gemini File API 업로드                   │  │
│  │ → PROCESSING 상태 대기                     │  │
│  │ → 분석 완료 후 파일 삭제                     │  │
│  └───────────────────────────────────────────┘  │
│                                                │
│  전 과정 SSE 스트리밍 (실시간 진행률 전송)          │
│  type: "progress" → step, percent, detail      │
│  type: "result"   → 최종 분석 텍스트             │
└────────────────┬───────────────────────────────┘
                 │ SSE Stream
                 ▼
┌─ 브라우저 ──────────────────────────────────────┐
│  실시간 프로그레스 바 업데이트                      │
│  분석 결과 렌더링                                │
│  → Supabase에 히스토리 자동 저장                   │
└─────────────────────────────────────────────────┘
```
