# AI 기획 자동화 기능 — 사전 기획 보고서

> 작성: 2026-05-01
> 대상: 운영/기획팀에서 요청한 9개 자동화 기능
> 목적: 강사·PM 미팅에서 받을 **레퍼런스 자료가 도착하기 전에** 우리가 어떤 인프라/구조/데이터를 갖춰야 하는지 미리 합의
> 상태: **의사결정용 보고서. 코드 변경 없음.**

---

## 0. 한 줄 요약

요청된 9개 기능은 본질적으로 **"강사별 컨텍스트(녹음본·메모·자료) + 레퍼런스 풀 + 프롬프트 → LLM 호출 → 형식화된 출력"** 의 같은 파이프라인 위에서 출력 종류만 다른 것이다. 따라서 9개를 따로 짜지 말고, **공통 엔진 1개 + 출력별 프롬프트/템플릿 9개**로 가야 한다.

지금 빠진 핵심은 (1) **STT(녹음본→텍스트)**, (2) **노션·외부링크 자동 패치**, (3) **레퍼런스 라이브러리 구조**, (4) **각 출력의 평가/편집 UI** 4가지.

## 1. 9개 기능 분류와 패턴

| # | 기능 | 입력 | 출력 형태 | 난이도 | 공통화 가능? |
|---|---|---|---|---|---|
| 1 | 무료강의 당일 PPT 장표 + 멘트 | 강의 자료, 강사 톤, 기존 PPT | PPT 파일 + 슬라이드별 발표 멘트 | ★★★★★ | 부분 |
| 2 | 무료강의 단톡방 공지 자동화 | 강의 일정, 강사명, 직전 알림 패턴 | 시간대별 공지 멘트 N개 | ★★ | ✅ |
| 3 | 미팅 녹음본 → 정리 봇 | mp3/m4a/노션링크 | 요약 + 액션아이템 + 결정사항 | ★★ | ✅ |
| 4 | 강의 PPT 기획안 초안 봇 | 미팅 정리, 강의 주제, 분량 | 슬라이드별 outline (텍스트) | ★★★ | ✅ |
| 5 | 알림톡/채널톡 멘트 자동화 | 강의 정보, 발송 목적, 톤 | 알림톡 변수 채운 텍스트 | ★★ | ✅ |
| 6 | 바이럴 질문 멘트 작성 봇 | 강의 주제, 타깃 페르소나 | 단톡방 질문 멘트 N개 | ★★ | ✅ |
| 7 | 붐업 멘트 (다양한 스타일) | 강의 주제, 분위기 톤 | 붐업 멘트 N개 (스타일별) | ★ | ✅ |
| 8 | 전자책 기획안 자동화 | 강사 강의/자료, 타깃, 분량 | 목차 + 챕터별 요약 | ★★★ | ✅ |
| 9 | 무료 상페 초안 기획안 | 강의 정보, 강사 스토리, 후기 | 섹션별 카피라이팅 | ★★★ | ✅ |

**1번(PPT 장표 자동화)을 제외한 8개는 같은 엔진으로 커버된다.** 출력이 텍스트(또는 마크다운/JSON)이고 LLM 한 번 호출이면 끝나기 때문. 1번만 PPT 파일 생성/편집(`pptx` 라이브러리)이라는 추가 레이어가 필요함.

## 3. 빠진 인프라 (이걸 먼저 만들어야 함)

### 3-1. STT (Speech-to-Text) — **가장 시급**
미팅 녹음본을 텍스트화하지 못하면 3, 4, 8, 9번이 사실상 불가능.

**옵션 비교**

| 옵션 | 비용 | 품질(한국어) | 한도 | 비고 |
|---|---|---|---|---|
| OpenAI Whisper API | $0.006/분 | ★★★★ | 25MB/file | 가장 무난, 1시간 = $0.36 |
| Google Gemini 1.5 Pro (audio 직접 입력) | $1.25/M tokens | ★★★★ | 1시간 audio = ~30만 토큰 | 이미 백엔드에 Gemini 붙어있음 — 추가 키 없이 재사용 가능 |
| ElevenLabs Scribe | $0.40/시간 | ★★★★★ | 무제한 | 한국어 화자 분리 강함, 비쌈 |
| 로컬 Whisper (HuggingFace Space) | 무료 | ★★★ | Space 절전 이슈 | 이미 백엔드 있어 확장 가능, 안정성 ↓ |

**1차 권장**: Gemini로 통일 — 이미 키/백엔드 있고, 1시간 미팅이 ~$0.4 수준이며, 한국어/화자 식별/타임스탬프 모두 OK.
**대안**: 정확도 더 필요하면 Whisper API 추가.

### 3-2. 노션 링크 자동 패치
사용자가 "미팅 정리 노션 링크"를 첨부 링크로 넣을 거라고 했음. 지금은 URL만 저장될 뿐 **내용을 안 읽음**. → LLM에 넘기려면 텍스트화 필요.

**옵션**:
- 노션 공식 API (`integrations/database/page.children`): 정식. 워크스페이스에 통합 추가만 하면 토큰 1개로 모든 페이지 fetch. **권장**.
- 공개 링크 fetch + HTML 파싱: 내부 비공개 페이지는 못 봄. 권장 X.

### 3-3. 레퍼런스 라이브러리 — **이게 핵심**

사용자 말처럼 "기획"이 붙는 작업은 AI가 기준점이 필요하다. **모범 사례 데이터셋**이 없으면 AI 출력은 항상 일반론에 그친다.

제안 구조:

```
/references/
├── alimtalk-good/         (지난 알림톡 중 반응 좋았던 것 - 메타데이터: 오픈율, 강사, 톤)
├── channel-talk-good/     (채널톡 답변 모범 사례)
├── viral-questions/       (단톡방 바이럴 질문 잘 된 케이스)
├── boom-up/               (붐업 멘트 스타일별: 친근/전문/감성/유머)
├── ppt-outlines/          (강사별 좋았던 PPT 구조)
├── ppt-master-decks/      (1번 기능용 — 직접 학습할 마스터 PPT 파일)
├── ebook-toc/             (좋은 전자책 목차 사례)
├── sales-pages/           (전환율 좋았던 무료 상페 카피)
└── _meta/
    └── index.json         (각 레퍼런스의 메타: 강사, 날짜, 성과 지표, 태그)
```

**저장 위치**: 자체 Supabase에 `references` 테이블 신설. 본문은 텍스트, 파일은 Supabase Storage. 각 레퍼런스에 (1) 카테고리, (2) 메타(누가/언제/성과), (3) 태그(분야: 부업/투자/육아 등 N잡연구소 분야 분류)를 붙임.

**RAG 사용 패턴**: 출력 생성 시 같은 카테고리 + 같은 분야 태그 레퍼런스 N개를 프롬프트에 few-shot으로 주입. 임베딩 검색까지 갈 필요 없음 — 카테고리·태그 필터로 충분.

### 3-4. 프롬프트/템플릿 관리

9개 기능별 프롬프트를 코드에 박지 말고 DB에 저장. 이유:
- 프롬프트는 자주 튜닝됨. 배포 없이 수정 가능해야 함.
- 강사·분야별 프롬프트 변형이 필요해질 가능성 높음.
- 결과 평가/A-B 테스트가 쉬워짐.

제안 테이블: `ai_prompts(id, feature_key, version, prompt_md, variables_schema, active, created_at)`. UI는 어드민 전용 1개 페이지로 해결.

### 3-5. 출력 편집·승인 UI

기획 출력은 **AI가 100% 자동 발송하면 안 된다.** 무조건 사람이 검토 → 수정 → 승인 → 발송 흐름이어야 한다 (이미 슝 알림톡이 그렇게 가고 있음 — 같은 패턴 유지).

UI 컴포넌트: `<AiDraftEditor>` — 좌측 입력/컨텍스트, 중앙 AI 출력(편집 가능), 우측 액션(재생성/대안 보기/발송 또는 복사). 9개 기능 모두 같은 컴포넌트 재사용.

## 4. 권장 빌드 순서 (3단계)

### Phase 0 — 기반 다지기 (레퍼런스 도착 전 / 지금 할 수 있는 것)

| 작업 | 산출물 | 예상 |
|---|---|---|
| `references` 테이블 + 어드민 업로드 UI | 사용자가 자료 도착 시 바로 분류해 넣을 수 있는 그릇 | 1일 |
| `ai_prompts` 테이블 + 어드민 편집 UI | 프롬프트 핫스왑 가능 | 0.5일 |
| `<AiDraftEditor>` 공통 컴포넌트 | 9개 기능 UI 통일 | 0.5일 |
| 노션 API 통합 (`/api/integrations/notion/page`) | 링크 첨부 시 자동 텍스트화 | 0.5일 |
| Gemini STT 어댑터 (`/api/integrations/transcribe`) | 음성 첨부 시 자동 텍스트화 | 1일 |
| `lecture-analyze-gemini` HuggingFace 의존 정리 | 절전 이슈 해결 (3-1 결정에 따라 자체 백엔드 제거 가능) | 0.5일 |

→ Phase 0 합계 약 **4일**. 레퍼런스 자료 도착 전에 완료 가능.

### Phase 1 — 쉬운 4개 먼저 (빠른 피드백 루프 확보)

순서 추천: **3 → 7 → 6 → 5**

1. **미팅 녹음본 정리 봇 (3번)** — 가장 단순. STT + 요약 프롬프트만. 이게 되면 강사 미팅마다 자동으로 텍스트가 쌓이기 시작 → 다른 기능들의 입력 데이터가 자연 축적됨.
2. **붐업 멘트 (7번)** — 레퍼런스 파일만 있으면 30분이면 끝. 결과물 보고 프롬프트 튜닝하기 좋음.
3. **바이럴 질문 (6번)** — 7번과 동형, 톤만 다름.
4. **알림톡/채널톡 멘트 (5번)** — 슝 발송 인프라 위에 멘트 생성만 추가. 발송 검증된 시스템에 합류.

→ Phase 1 합계 약 **3~4일**. 레퍼런스가 어느 정도 들어온 시점.

### Phase 2 — 기획 초안류 3개 (출력이 길고 구조적인 것)

순서 추천: **4 → 9 → 8**

5. **강의 PPT 기획안 초안 (4번)** — 마크다운 outline 출력. 사용자가 받아서 PPT로 직접 옮김.
6. **무료 상페 초안 (9번)** — 섹션별 카피라이팅. 헤드라인/소제목/본문/CTA 분리.
7. **전자책 기획안 (8번)** — 가장 긴 출력. 챕터 단위로 나눠 호출(컨텍스트 절약).

→ Phase 2 합계 약 **4~5일**.

### Phase 3 — PPT 자동 생성 (1번, 별도 트랙)

가장 어려움. 별도로 다룸 ↓

| 단계 | 내용 | 비고 |
|---|---|---|
| 3-1 | 마스터 PPT 5~10개 수집 + 슬라이드 단위로 분해 | 사용자 수동 |
| 3-2 | 슬라이드 분류기 (intro/agenda/case-study/cta…) | 수동 라벨 → 향후 LLM 자동 분류 |
| 3-3 | "강의 정보 → 슬라이드 시퀀스 outline" 생성 (4번 결과 재사용) | LLM |
| 3-4 | outline → `pptxgenjs` 또는 `python-pptx`로 PPT 빌드 | 디자인 템플릿 1~3종 고정 |
| 3-5 | 슬라이드별 발표 멘트(노트) 생성 | LLM, 슬라이드 본문 + 강사 톤 입력 |

**현실적 권장**: Phase 3-1~3-3까지만 자동화하고 outline 텍스트를 PM에게 넘기는 형태로 시작. 실제 PPT 파일 빌드는 디자인 일관성 위험이 커서 다음 분기에 검토.

### "단톡방 공지 자동화 (2번)"는 어디?

이건 **AI 생성 비중이 작고 스케줄링·트리거가 핵심**. Phase 1과 별개 트랙으로 다룸:

- 입력: 무료강의 시작 일정 (DB에서 조회), 강사명, 채널톡방 ID
- 트리거: T-24h, T-1h, T-10min, T+0, T+1h 같은 시점
- AI 역할: 시점·강사 맞춤 멘트 1줄 생성 (붐업 톤 적용 — 6/7번과 같은 라이브러리 사용)
- 발송: 채널톡 / 카카오 단톡방 봇 (별도 인프라 필요. 카톡 단톡방 자동발송은 카카오 정책상 비공식 → 채널톡으로 한정 권장)

→ 약 **1~1.5일**. Phase 1 마치고 6/7번 멘트 라이브러리 쌓인 다음에.

## 5. 강사로부터 받아야 할 자료 체크리스트

레퍼런스 자료 요청 시 누락 없도록 정리. 사용자가 강사·PM에게 요청할 때 그대로 사용 가능.

### 공통 (모든 강사 1회씩)
- [ ] 미팅 녹음본 (mp3/m4a, 1시간 이내 권장)
- [ ] 미팅 정리 노션 페이지 링크 (공유 권한 — 통합 토큰으로 접근)
- [ ] 강사 자기소개 / 톤 샘플 (블로그 글, 영상 자막, SNS 캡션 — 어떤 거든 1000자 이상)
- [ ] 강사 분야 태그 (부업/투자/육아/디자인/AI/...)
- [ ] 타깃 페르소나 (1~2명, 나이·직업·고민 수준)

### 기능별 (해당하는 것만)

| 기능 | 추가 자료 |
|---|---|
| 5/6/7 (멘트류) | 지난 알림톡 / 붐업 멘트 중 반응 좋았던 것 5~10개 (스크린샷 또는 텍스트) |
| 4 (PPT outline) | 비슷한 주제의 좋은 PPT 1~2개 (PDF 또는 .pptx) |
| 1 (PPT 자동) | 마스터 PPT 템플릿 5~10개 + 디자인 가이드 (폰트/컬러) |
| 8 (전자책) | 좋아하는 전자책 1~2개 목차 + 분량(페이지) 목표 + 형식(어조: 격식/반말) |
| 9 (상페) | 전환율 좋았던 상페 URL 3~5개 + 후기 텍스트 |

레퍼런스 메타 입력 시 **"왜 이게 좋았나"** 한 줄 같이 받기. AI가 그대로 따라 하지 않고 "좋은 이유"를 살리도록 프롬프트에 인용함.

## 9. 한 페이지 요약 (의사결정자용)

- 9개 중 8개는 **공통 엔진 1개 + 출력 템플릿 9개**로 끝남.
- 가장 어려운 1번(PPT 자동 생성)은 **outline까지만 자동, PPT 빌드는 다음 분기**로 분리 권장.
- **빠진 인프라 4가지**: STT(Gemini 권장), 노션 패치, 레퍼런스 DB, 출력 편집 UI. 약 4일 작업.
- **순서**: 인프라 4일 → 멘트 4개(3·5·6·7) 4일 → 기획 3개(4·8·9) 5일 → 단톡방 자동화(2) 1.5일 → PPT(1) 별도 분기.
- **레퍼런스 자료 요청 체크리스트** 5장 1번에 정리 — 사용자가 그대로 강사/PM에게 전달 가능.
- **비용 ~$10/월**, 무시 가능 수준.

이 보고서에 동의·수정사항 주시면 Phase 0 작업부터 들어가겠습니다.
