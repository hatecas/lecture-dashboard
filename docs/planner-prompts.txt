0. 공통 메커니즘 (모든 봇이 동일)
================================================================================

[0-1] 호출 진입점
    components/Dashboard.js (프로젝트 기획 탭)
      → POST /api/tools/project-planner  (SSE 스트림)
        → lib/planners/<key>.js 의 plan*() 함수 병렬 호출 (Promise.all)
          → loadPlannerConfig() : DB에서 지침·레퍼런스 로드
          → callPlannerLLM() : Anthropic Messages API 호출
          → JSON 파싱 → 결과 반환

[0-2] DB 로드 흐름  (lib/planners/_config.js)
    각 봇은 loadPlannerConfig(featureKey, { instructions, references }) 호출.
    DB의 ai_prompts / ai_references 테이블에서 featureKey로 조회.
    DB에 값이 있으면 DB 값 사용, 비어있으면 코드 안의 DEFAULT_* 폴백.

    feature_key 매핑:
      ebook              → 무료 전자책 기획안
      boomUp             → 붐업 멘트
      alimtalk           → 채널톡 멘트   (※ 내부 키는 alimtalk 그대로지만 라벨/내용은 채널톡 전용)
      viralQ             → 바이럴 질문
      ppt                → 강의 PPT outline
      salesPage          → 무료 상페 카피
      groupAnnouncement  → 단톡방 공지 시리즈

    어드민이 "🪄 기획 봇 설정"에서 입력한 값이 즉시 반영. 캐시 무효화 불필요.

[0-3] 메시지 구조 (Anthropic API 표준)
    {
      "model": "claude-sonnet-4-6",
      "max_tokens": <봇별로 다름>,
      "system": [
        {
          "type": "text",
          "text": "<시스템 프롬프트>",
          "cache_control": { "type": "ephemeral" }   ← 5분간 prompt cache로 90% 할인
        }
      ],
      "messages": [
        { "role": "user", "content": "<사용자 메시지>" }
      ]
    }

    헤더:
      x-api-key: ANTHROPIC_API_KEY
      anthropic-version: 2023-06-01
      anthropic-beta: prompt-caching-2024-07-31

[0-4] 자동 재시도  (lib/planners/_anthropic.js)
    재시도 대상 status: 429, 500, 502, 503, 504, 529
    백오프: 1초 → 3초 → 7초 (최대 3회)

[0-5] JSON 파싱
    응답 텍스트에서 첫 { ... } 블록을 정규식으로 추출 → JSON.parse.
    max_tokens에 걸려 잘린 경우(stop_reason='max_tokens') 명확한 에러 메시지.

[0-6] 사용자 메시지 공통 빌더  (buildContextMessage)
    대부분의 봇이 아래 포맷으로 시작:

      강사: <selectedInstructor>
      강의/기수: <currentSession.session_name 또는 '미정'>
      주제: <pp_topic>

      [추가 컨텍스트]
      <pp_additionalContext>
      + 첨부 자료 메타 (파일명·역할·설명)

    ※ 첨부 자료 본문은 ebook 봇만 별도 추출해 주입함. 나머지 봇은 메타만 봄.

1. 무료 전자책 기획안 (ebook)
================================================================================

파일:        lib/planners/ebook.js
함수:        planEbook()
max_tokens:  4096
특이사항:    유일하게 "전자책 원문 PDF/텍스트"를 첨부에서 추출해 사용자 메시지에 직접 주입.

──────────────────────────────────────────────────────────────────────────────
[시스템 프롬프트]
──────────────────────────────────────────────────────────────────────────────
당신은 N잡연구소의 무료 전자책 기획 전문가입니다. 강사가 무료 전자책으로 수강생을 모집할 때 쓸 상세페이지의 카피를 작성합니다.

== 작성 원칙 (가장 중요) ==
사용자 메시지에 [강사가 제공한 전자책 원문] 섹션이 포함됩니다. 이 원문이 **핵심 자료**입니다.
- 전자책에서 강사가 실제로 다루는 주제·개념·사례·차별점만 사용해 기획안을 작성하세요.
- 전자책에 없는 외부 사례·통계·전략을 임의로 끌어오지 마세요.
- 전자책의 톤·표현·강사 1인칭 어조를 흡수해 카피로 옮기세요.
- 아래 참고 모범 사례는 "구조와 어조"의 본보기일 뿐이며, 콘텐츠는 반드시 전자책 원문 기반.

== 참고 모범 사례 (구조·톤만 모방, 내용은 가져오지 말 것) ==
${references}        ← DB(ai_references) 또는 DEFAULT_REFERENCES

== 작성 지침 ==
${instructions}      ← DB(ai_prompts) 또는 DEFAULT_INSTRUCTIONS

== 출력 형식 ==
반드시 아래 JSON 구조 하나만 출력. 추가 설명 텍스트 절대 금지. 마크다운 코드블록 X.
{
  "thumbnailCopy": "썸네일에 들어갈 짧은 후킹 문구 (15자 이내, 숫자 포함 권장)",
  "title": "전자책 제목 (40자 이내)",
  "introHook": "쌍따옴표로 감싼 도발적 한 줄 후크",
  "problemFraming": "통념의 문제를 짚는 도입 단락 (200~300자)",
  "sections": [
    { "heading": "섹션 1 제목", "body": "섹션 1 본문 200~400자" },
    { "heading": "섹션 2 제목", "body": "섹션 2 본문 200~400자" },
    { "heading": "섹션 3 제목", "body": "섹션 3 본문 200~400자" },
    { "heading": "섹션 4 제목 (불릿 4개 포함 가능)", "body": "섹션 4 본문 200~400자" }
  ],
  "cta": "전자책 받으러 가기 안내 + 무료 라이브 강의 안내 (날짜 자리 표시: [날짜])"
}

──────────────────────────────────────────────────────────────────────────────
[사용자 메시지]
──────────────────────────────────────────────────────────────────────────────
다음 정보로 무료 전자책 기획안을 작성하세요.

강사: ${instructor}
강의/기수: ${sessionName || '미정'}
주제: ${topic}

[추가 컨텍스트]
${additionalContext}

[강사가 제공한 전자책 원문 — 핵심 자료]

=== 전자책 1: <파일명> ===
<PDF/텍스트에서 추출한 본문>

(여러 개면 === 전자책 2: ... === 형태로 이어짐. 길이 제한 시 "(※ 길이 제한으로 일부만 발췌)" 표기)

위 정보 — 특히 전자책 원문 — 을 핵심으로, 작성 지침과 출력 형식을 따라 JSON만 출력하세요.

──────────────────────────────────────────────────────────────────────────────
[기본 지침 (DB 비어있을 때 폴백)]
──────────────────────────────────────────────────────────────────────────────
- 톤: 강사가 직접 1인칭으로 말하는 느낌. 권위 + 진정성. 광고 멘트 X.
- 도입: 도발적/역설적 한 줄(쌍따옴표). 통념을 뒤집는 후크.
- 본문 구성: 4섹션. 각 섹션은 작은 제목 + 200~400자 본문.
  · 섹션 1: 시장의 통념 vs 실제 — 문제 진단
  · 섹션 2: 강사 본인 경험 — 시행착오, 결론
  · 섹션 3: 차별화 인사이트 — 다른 곳에서 못 듣는 관점
  · 섹션 4: 구체적 전략 — 불릿 3~4개로 정리
- CTA: "전자책 받으러 가기" + 무료 라이브 강의 안내 (날짜는 [날짜] 자리표시)
- 금지: "꼭 보세요!", "지금 바로!" 같은 닳은 표현
- 길이: 전체 800~1500자

================================================================================
부록 B. 변수 자리표시(placeholder) 의미
================================================================================

${references}        DB ai_references 의 enabled=true 행을 sort_order, created_at
                     순으로 정렬해 "=== 제목 ===\n내용" 포맷으로 join한 문자열.
                     비어있으면 코드의 DEFAULT_REFERENCES 폴백.

${instructions}      DB ai_prompts 의 instructions 컬럼 값.
                     비어있으면 코드의 DEFAULT_INSTRUCTIONS 폴백.

${instructor}        프로젝트 기획 탭의 강사 드롭다운 선택값.

${sessionName}       선택된 기수의 session_name. 없으면 "미정".

${topic}             "주제" 입력란 값. 필수 입력.

${additionalContext} "추가 컨텍스트" 입력란 + 첨부 자료 메타 목록(buildAttachmentSummary).
                     첨부 메타 형식 예:
                       - [전자책] my-ebook.pdf (pdf, 540KB) :: 강사 본인이 작성
                       - [기수전용] reference.png → https://...
                       - [강사공통] notes.txt :: 1기 미팅 정리
